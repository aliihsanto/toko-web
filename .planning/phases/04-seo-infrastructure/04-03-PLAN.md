    **Part D: Fix ScrollReveal CLS for above-fold content (SEO-06)**

    The current ScrollReveal wraps hero content with `initial="hidden"` (opacity:0, y:30), which causes:
    - CLS: Content starts invisible, then shifts when revealed
    - LCP delay: The h1 heading (often the LCP element) starts hidden

    Fix approach -- update ScrollReveal component to accept a `noTransform` prop:
    - Add optional prop `noTransform?: boolean` to ScrollRevealProps
    - When noTransform is true, use variants with only opacity (no y/x transform): { hidden: { opacity: 0 }, visible: { opacity: 1 } }
    - This eliminates the layout shift while preserving fade-in

    ALSO: In the homepage (page.tsx), remove ScrollReveal from the FIRST hero elements (the badge, h1, subtitle, CTA buttons). These should render immediately without animation for optimal LCP. Keep ScrollReveal on below-fold sections.
    - Remove ScrollReveal wrapping from: badge span, h1, subtitle p, CTA buttons div, stats strip (all in the hero section)
    - Keep ScrollReveal on: services section, trust signals, why us, sectors preview, global reach, CTA footer section

    In PageHero component: Remove ScrollReveal wrappers from h1, subtitle, and badge. These are always above-fold. Keep the WaveDivider.
