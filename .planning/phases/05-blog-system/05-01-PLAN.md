1. Install dependencies:
   ```bash
   npm install velite -D
   npm install @shikijs/rehype shiki rehype-slug rehype-autolink-headings remark-gfm
   ```

2. Create `velite.config.ts` at project root with:
   - `defineCollection` named 'Post' with pattern `'blog/**/*.mdx'`
   - Schema fields: `title` (s.string().max(120)), `description` (s.string().max(300)), `date` (s.isodate()), `published` (s.boolean().default(true)), `category` (s.string()), `tags` (s.array(s.string()).optional()), `image` (s.string().optional()), `imageAlt` (s.string().optional()), `slug` (s.path()), `body` (s.mdx()), `toc` (s.toc()), `metadata` (s.metadata()), `excerpt` (s.excerpt({ length: 200 }))
   - `defineConfig` with root: `'content'`, output.data: `'.velite'`, output.assets: `'public/static'`, output.base: `'/static/'`, output.name: `'[name]-[hash:6].[ext]'`, output.clean: true
   - mdx.remarkPlugins: [remarkGfm]
   - mdx.rehypePlugins: [rehypeSlug, [rehypeAutolinkHeadings, { behavior: 'wrap' }], [rehypeShiki, { theme: 'github-light' }]]
   - CRITICAL: rehype plugin order matters -- slug first, then autolink-headings, then shiki

3. Update `next.config.ts` to trigger Velite build:
   - Add BEFORE the withNextIntl call:
   ```typescript
   const isDev = process.env.NODE_ENV === 'development';
   const isBuild = process.argv.includes('build');
   if (!process.env.VELITE_STARTED && (isDev || isBuild)) {
     process.env.VELITE_STARTED = '1';
     import('velite').then((m) => m.build({ watch: isDev, clean: !isDev }));
   }
   ```
   - CRITICAL: Do NOT use VeliteWebpackPlugin (Turbopack default in Next.js 16 ignores webpack plugins)
   - CRITICAL: Do NOT use `process.argv.includes('dev')` -- broken in Next.js 16. Use NODE_ENV instead.

4. Update `tsconfig.json` compilerOptions.paths to add:
   ```json
   "#site/content": ["./.velite"]
   ```
   Keep existing `"@/*": ["./src/*"]` path intact.

5. Update `.gitignore` to add:
   ```
   # velite
   .velite
   ```

6. Update `vitest.config.ts` resolve.alias to add the `#site/content` alias:
   ```typescript
   '#site/content': path.resolve(__dirname, './.velite'),
   ```

7. Create empty `content/blog/tr/` directory (and en/, fr/, ru/) so Velite has something to scan. Create a minimal placeholder MDX file `content/blog/tr/_placeholder.mdx` with `published: false` so Velite generates the .velite output directory structure. This placeholder will be removed in Plan 03 when real seed content is added.
