  <action>
    **All form components follow the SAME pattern** (from research Pattern 3):
    1. `'use client'` directive
    2. `useForm<SchemaType>({ resolver: zodResolver(schema), defaultValues: {...} })`
    3. `useActionState(serverAction, { success: false, message: '' })` for server submission
    4. `useGoogleReCaptcha()` hook for reCAPTCHA token
    5. `onSubmit` callback: get reCAPTCHA token via `executeRecaptcha('form_name')`, construct FormData from validated data, append recaptchaToken, call formAction
    6. Render form with `handleSubmit(onSubmit)` on the form element's onSubmit
    7. Use `useTranslations('Forms.contact')` for field labels and `useTranslations('Validation')` for error messages
    8. If `state.success` is true, render `<FormSuccess message={state.message} />` instead of the form
    9. If `state.message` exists and `!state.success`, show error alert above the form
    10. Include `<HoneypotField register={register} />` inside the form
    11. Include `<SubmitButton pending={pending}>` at the bottom

    **IMPORTANT per research Pitfall 7**: Do NOT put the Server Action on the form's `action` attribute. Use react-hook-form's `handleSubmit` for client validation, then manually call the formAction function from useActionState. The form element should use `onSubmit={handleSubmit(onSubmit)}` NOT `action={formAction}`.

    **IMPORTANT per research Pitfall 1**: Call `executeRecaptcha` inside `onSubmit`, NOT on mount. Tokens expire after 2 minutes.

Map zod error types to translation keys: use `tValidation('fieldName.errorType')` pattern where errorType comes from the zod error. For simplicity, use a helper function that checks `errors[field]?.message` and returns a translated string based on the error type (too_small, invalid_string, etc.).
