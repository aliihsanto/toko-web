    **Step 3: Create Zod schemas** (language-agnostic, no hardcoded error messages)

    `src/lib/schemas/contact.ts`: Export `contactSchema` and `ContactFormData` type.
    Fields: name (string, min 2, max 100), email (string, email), subject (string, min 3, max 200), message (string, min 10, max 5000), website (string, max 0, optional -- honeypot).

    `src/lib/schemas/quote.ts`: Export `quoteSchema` and `QuoteFormData` type.
    Fields: name (string, min 2, max 100), email (string, email), company (string, optional, max 200), product (string, min 3, max 300), quantity (string, min 1, max 100), destinationCountry (string, min 2, max 100), details (string, optional, max 5000), website (string, max 0, optional -- honeypot).

    `src/lib/schemas/sourcing.ts`: Export `sourcingSchema` and `SourcingFormData` type.
    Fields: name (string, min 2, max 100), email (string, email), company (string, optional, max 200), product (string, min 3, max 300), specifications (string, min 10, max 5000), quantity (string, min 1, max 100), targetCountry (string, min 2, max 100), website (string, max 0, optional -- honeypot).

    `src/lib/schemas/callback.ts`: Export `callbackSchema` and `CallbackFormData` type.
    Fields: name (string, min 2, max 100), phone (string, regex for international phone: `/^\+?[0-9\s\-\(\)]{7,20}$/`), preferredTime (string, min 1, max 100), subject (string, min 3, max 200), website (string, max 0, optional -- honeypot).

    **Step 4: Create shared FormState type**

    `src/lib/actions/types.ts`:
    ```typescript
    export type FormState = {
      success: boolean;
      message: string;
      errors?: Record<string, string[]>;
    };
    ```

    **Step 5: Create rate limiter** (`src/lib/rate-limit.ts`)
    Implement sliding window rate limiter using a Map. WINDOW_MS = 60000 (1 min), MAX_REQUESTS = 5. Include cleanup when map exceeds 10000 entries. Export `rateLimit(identifier: string): { success: boolean }`. Follow the exact pattern from the research document.

    **Step 6: Create reCAPTCHA verifier** (`src/lib/recaptcha.ts`)
    Server-side verification against `https://www.google.com/recaptcha/api/siteverify`. SCORE_THRESHOLD = 0.5. If `RECAPTCHA_SECRET_KEY` is not set, log warning and return true (development mode). Export `verifyRecaptcha(token: string): Promise<boolean>`. Follow the exact pattern from research.

    **Step 7: Create Resend email client** (`src/lib/email/resend.ts`)
    Create Resend instance with `process.env.RESEND_API_KEY`. Define TOKO_EMAIL = 'info@toko.com.tr', FROM_ADDRESS = 'Toko Website <noreply@toko.com.tr>'. Export `sendNotificationEmail(type, data)` that dynamically selects the email template based on type ('contact' | 'quote' | 'sourcing' | 'callback'), generates subject line, and sends via `resend.emails.send()` with React Email template. If RESEND_API_KEY is not set, log warning and skip (development mode).

    **Step 8: Create email templates** (4 files in `src/lib/email/templates/`)
    Each template is a React component using @react-email/components (Html, Head, Body, Container, Section, Heading, Text, Hr). Clean, professional layout showing all submitted fields. Each template receives the typed form data as props.

    - `contact-notification.tsx`: Shows name, email, subject, message
    - `quote-notification.tsx`: Shows name, email, company, product, quantity, destination country, details
    - `sourcing-notification.tsx`: Shows name, email, company, product, specifications, quantity, target country
    - `callback-notification.tsx`: Shows name, phone, preferred time, subject

    **Step 9: Create Server Actions** (4 files in `src/lib/actions/`)
    Each action follows the SAME pattern from research Pattern 2:
    1. Mark with `'use server'`
    2. Signature: `(prevState: FormState, formData: FormData) => Promise<FormState>`
    3. Get IP from `headers()` x-forwarded-for
    4. Check rate limit -> return error if exceeded
    5. Check honeypot (website field) -> return fake success if filled
    6. Verify reCAPTCHA token -> return error if failed
    7. Parse form data with zod safeParse -> return field errors if invalid
    8. Call sendNotificationEmail -> return error if email fails
    9. Return success message
    Use `getTranslations('Forms')` from next-intl/server for translated messages (keys: `errors.rateLimited`, `errors.recaptchaFailed`, `errors.validationFailed`, `errors.sendFailed`, `success`).

    Files: `src/lib/actions/contact.ts`, `src/lib/actions/quote.ts`, `src/lib/actions/sourcing.ts`, `src/lib/actions/callback.ts`. Each imports its own schema and calls `sendNotificationEmail` with its own type.

    **Step 10: Create shared form UI primitives**

    `src/components/forms/submit-button.tsx`: Client component. Accepts `pending: boolean` and `children: React.ReactNode`. Renders a Button (from shadcn) with loading spinner (Loader2 from lucide) when pending, disabled when pending. Style: rounded-full, bg-primary, text-white, shadow-md shadow-primary/20, full width.

    `src/components/forms/form-success.tsx`: Client component. Accepts `message: string`. Renders a success state with CheckCircle2 icon (lucide, text-emerald-600), the message text, and a Framer Motion scale-in animation. Use rich-card styling with border-t-4 border-t-emerald-500.

    `src/components/forms/honeypot-field.tsx`: Client component. Renders a hidden input field with name="website", aria-hidden="true", tabIndex={-1}, autoComplete="off", positioned absolutely -left-[9999px] opacity-0. Accepts `register` function from react-hook-form to connect the field.
